# Техническое задание  
## «Разработка шаблонного класса Matrix для линейной алгебры на C++23»

### 1. Назначение и область применения

Разработать шаблонный класс `Matrix<T, Rows, Cols, Allocator>`, предоставляющий эффективные и безопасные операции с двумерными матрицами. Класс должен поддерживать:

- статически и динамически определённые размеры;
- арифметические операции (поэлементные, матричные, скалярные);
- основные алгоритмы линейной алгебры (транспонирование, детерминант, обращение);
- совместимость с STL (итераторы, аллокаторы, концепты);
- высокую производительность (expression templates, кэш-локальность, опционально SIMD).

### 2. Общие требования

| Параметр | Значение |
|----------|----------|
| Язык | C++23 (с учётом обратной совместимости с C++20) |
| Пространство имён | `math` (или `linalg`) |
| Заголовочные файлы | `matrix.hpp`, `matrix_impl.hpp` (опционально) |
| Модули (C++20) | Поддержка модулей: `export module math.matrix;` |
| Исключения | Разрешены, но критические операции (`swap`, перемещение) — `noexcept` |
| Многопоточность | Чтение из разных потоков — безопасно, мутация — требует внешней синхронизации |

### 3. Параметризация и работа с памятью

#### 3.1. Параметры шаблона

```cpp
template<
    typename T,
    std::size_t Rows = Dynamic,   // Dynamic = static constexpr std::size_t(-1)
    std::size_t Cols = Dynamic,
    typename Allocator = std::pmr::polymorphic_allocator<T>
>
class Matrix;
```

- **`T`** — арифметический тип (`std::is_arithmetic_v<T>`), требование enforced через концепт C++20.
- **`Rows`, `Cols`** — размерность. Если `Dynamic` — размер задаётся в конструкторе и может меняться (через `resize`).  
  Поддержка смешанного режима: `Matrix<float, 3, Dynamic>`.
- **`Allocator`** — соответствует модели `Allocator` C++20. По умолчанию — полиморфный аллокатор из `std::pmr`.

#### 3.2. Хранение данных

- **Статическая размерность** (`Rows != Dynamic && Cols != Dynamic`):  
  Внутренний тип хранения — `std::array<T, Rows * Cols>` или выровненный массив `alignas(64) T data[Rows*Cols]`.  
  Запрещено динамическое выделение памяти.
- **Динамическая размерность** (хотя бы один `Dynamic`):  
  Использовать `std::vector<T, Allocator>` (или собственную обёртку с учётом выравнивания для SIMD).  
  Поддержка `shrink_to_fit()` и `reserve()`.

#### 3.3. Layout

Строко-мажорный порядок (row-major) по умолчанию. Допускается переключение на колонко-мажорный через политику (отдельный необязательный шаблонный параметр `Layout = RowMajor`).

### 4. Конструкторы и присваивание

Класс должен предоставлять:

| Конструктор / оператор | Требования |
|------------------------|-------------|
| `Matrix() = default` | Для статически размеренных матриц создаёт нулевую (value-initialized). Для динамических — матрицу 0x0. |
| `Matrix(size_t rows, size_t cols)` | Для динамических размеров. Выделяет память, элементы инициализированы нулём. |
| `Matrix(std::initializer_list<std::initializer_list<T>>)` | Построение из вложенных списков. Проверка согласованности размеров. |
| `Matrix(const Matrix&)` / `Matrix(Matrix&&)` | Стандартные, с поддержкой аллокаторов (propagation). |
| `operator=(const Matrix&)` / `operator=(Matrix&&)` | Копирование/перемещение с учётом возможного изменения размеров (для динамических). |
| `template<U> Matrix(const Matrix<U, R2, C2>&)` | Конвертирующий конструктор, если `U` конвертируется в `T` и размерности совместимы. |

### 5. Доступ к элементам

#### 5.1. Основные операторы

```cpp
T& operator()(size_t i, size_t j);
const T& operator()(size_t i, size_t j) const;
T& operator[](size_t i) const;          // возвращает прокси-строку с operator[]
```

- `operator()` — без проверки границ в release, с `assert` в debug.
- `at(i, j)` — с проверкой границ, бросает `std::out_of_range`.

#### 5.2. Прямой доступ к данным

- `T* data()` — указатель на начало (строка-мажор).
- `std::span<T> row(size_t i)` — невладеющее представление строки (C++20).
- `const std::span<const T> row(size_t i) const`.

### 6. Итераторы и совместимость с STL

#### 6.1. Основные итераторы

- `iterator`, `const_iterator` — обход всех элементов в порядке row-major.
- `begin()`, `end()`, `cbegin()`, `cend()`.
- Итераторы должны быть категории `random_access_iterator`.
- `reverse_iterator` через `rbegin()`, `rend()`.

#### 6.2. Дополнительные обзоры

- `row_iterator` / `column_iterator` — итераторы по строкам/столбцам (возвращают `std::span`).
- Поддержка диапазонов (C++20 ranges): метод `rows()` возвращает `std::ranges::views::all` строк.

### 7. Математические операции

#### 7.1. Поэлементные и скалярные операции

```cpp
Matrix& operator+=(const Matrix&);
Matrix& operator-=(const Matrix&);
Matrix& operator*=(const T& scalar);
Matrix& operator/=(const T& scalar);
friend Matrix operator+(Matrix, const Matrix&);   // через +=
friend Matrix operator-(Matrix, const Matrix&);
friend Matrix operator*(Matrix, const T&);
friend Matrix operator*(const T&, Matrix);
friend Matrix operator/(Matrix, const T&);
```

#### 7.2. Матричное умножение

```cpp
template<size_t R2, size_t C2>
Matrix<T, Rows, C2> operator*(const Matrix<T, R2, C2>& other) const;
```

- Требование: `Cols == R2` или `Cols == Dynamic && R2 == Dynamic` (проверка во время выполнения).
- Для больших матриц (>64x64) реализовать блочное умножение (кэш-оптимизированное).  
- Для динамических матриц — возможность подключения внешнего BLAS через compile-time флаг.

#### 7.3. Транспонирование

- `Matrix<T, Cols, Rows> transpose() const & noexcept` — возвращает новую матрицу.
- Для статически размеренных — `constexpr`.

#### 7.4. Линейная алгебра (квадратные матрицы)

```cpp
T determinant() const requires (Rows == Cols);
Matrix inverse() const requires (Rows == Cols);
T trace() const requires (Rows == Cols);
```

- Вычисление детерминанта через LU-разложение с частичным выбором ведущего элемента.
- Обратная матрица — через решение системы `A * X = I`.
- В случае вырожденности — бросать `std::domain_error`.

#### 7.5. Разложения (опционально)

Функции свободные или статические методы:
- `LU`, `QR`, `Cholesky` (для симметричных положительно определённых).
- Каждое разложение — отдельный класс с методами `solve()`, `det()` и т.д.

### 8. Статические фабричные методы

```cpp
static Matrix zeros();            // все элементы нули
static Matrix ones();             // все элементы единицы
static Matrix identity();         // единичная матрица (квадратная)
static Matrix fill(const T& value);
static Matrix from_diag(std::initializer_list<T> diag); // диагональная
```

### 9. Сравнение и совместимость с C++20

```cpp
bool operator==(const Matrix& other) const;
auto operator<=>(const Matrix& other) const;  // лексикографическое сравнение
```

- Для типов с плавающей запятой — сравнение с задаваемой пользователем точностью (статическая переменная `epsilon` или параметр метода).

### 10. Проверки и безопасность

#### 10.1. Контракты и asserts

- В debug-режиме все индексы проверяются через `assert`.
- В release режиме `operator()` — без проверки (максимальная скорость), `at()` — всегда проверяет.

#### 10.2. Обработка ошибок

- Несовместимые размеры в операциях (сложение, умножение) — `std::invalid_argument`.
- Выход за границы в `at()` — `std::out_of_range`.
- Вычисление обратной/детерминанта для неквадратной — `static_assert` или `std::logic_error` (для динамических).

#### 10.3. noexcept спецификации

- Конструктор перемещения, `swap()`, `transpose()` (если не выделяет память) — `noexcept`.
- `rows()`, `cols()`, `data()` — `noexcept`.

### 11. Производительность и оптимизации

#### 11.1. Expression Templates (обязательно для поэлементных операций)

- Определить прокси-типы `MatrixSum`, `MatrixScalarProduct` и т.д., лениво вычисляемые.
- Операторы `+`, `-`, `*` (поэлементный) должны возвращать выражения, а не материализованные матрицы, если не присваивается результат.
- При присваивании (через `operator=`) выполнять свёртку в один цикл.

#### 11.2. SIMD и выравнивание

- Для статических матриц размером, кратным 4/8, выравнивание на 32 или 64 байта (`alignas`).
- Для динамических — использование `std::aligned_alloc` или кастомного аллокатора с выравниванием.
- Опционально: внутренние циклы, помеченные `#pragma omp simd` или `[[gnu::simd]]`.

#### 11.3. Кэш-локальность

- Обход в операциях производить с шагом 1 по памяти (row-major).
- Умножение — блочная версия с размером блока, подобранным под размер кэш-линии.

### 12. Сериализация и ввод/вывод

#### 12.1. Текстовый формат

```cpp
friend std::ostream& operator<<(std::ostream&, const Matrix&);
friend std::istream& operator>>(std::istream&, Matrix&);
```

- Формат вывода: каждая строка на отдельной строке, элементы разделены пробелом/табуляцией.
- Ввод: чтение до конца строки, определение размеров автоматически (для динамических).

#### 12.2. Бинарная сериализация

```cpp
void write_binary(std::ostream&) const;
void read_binary(std::istream&);
```

- Записывает размеры (`uint64_t`, `uint64_t`), затем сырые данные типа `T`.

#### 12.3. Поддержка `std::format` (C++20/23)

- Специализация `std::formatter<Matrix<T,Rows,Cols>>` для форматированного вывода.

1. Класс компилируется без предупреждений с флагами `-Wall -Wextra -pedantic -std=c++23`.
2. Все методы корректно обрабатывают крайние случаи (пустые матрицы, вырожденные и т.д.).
3. Производительность умножения на матрицах 512x512 не более чем в 1.5 раза медленнее Eigen (при отключении expression templates Eigen).
